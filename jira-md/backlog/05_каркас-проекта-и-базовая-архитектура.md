# [SWIFTMIG-4] Каркас проекта и базовая архитектура
Настройка монорепозитория SwiftMigrator: SPM-пакет MigratorKit (MigratorCore, MigratorCLI, тесты) + Xcode-проект с app-таргетом MigratorUI.
Результат эпика: компилирующийся скелет всех трёх таргетов, CI, доменная модель и протоколы слоёв, на которые лягут этапы 1-3 плана диплома.
Оценка: 6-9 рабочих дней.

## [SWIFTMIG-5] Создать структуру репозитория и таргеты
> estimate: 1д
> labels: setup

Создать монорепозиторий: локальный SPM-пакет MigratorKit (таргеты MigratorCore, MigratorCLI executable, MigratorCoreTests) и Xcode-проект SwiftMigrator.xcodeproj с app-таргетом MigratorUI (macOS 14+, SwiftUI), подключающим MigratorCore как local package dependency.

Что сделать:
- Структура: /Package.swift, /Sources/MigratorCore, /Sources/MigratorCLI, /Tests/MigratorCoreTests, /App/SwiftMigrator.xcodeproj
- MigratorCore не импортирует SwiftUI/AppKit (только Foundation при необходимости)
- CLI собирается через swift build, app - через xcodebuild

Критерии приёмки:
- swift build && swift test проходят из корня
- Приложение запускается и показывает пустое окно
- В MigratorCore нет import UI-фреймворков (проверено grep)

## [SWIFTMIG-6] Подключить и зафиксировать зависимости
> estimate: 1д
> labels: setup

Добавить внешние зависимости с зафиксированными версиями и проверить smoke-тестами. Версии пиновать: swift-syntax обязан соответствовать версии тулчейна.

Что сделать:
- swift-syntax (версия под текущий тулчейн Xcode) в MigratorCore
- XcodeProj (tuist) в MigratorCore
- swift-argument-parser в MigratorCLI
- Smoke-тесты: парсинг строки кода в SourceFileSyntax; открытие пустого .xcodeproj из фикстуры; --help у CLI
- Записать в DECISIONS.md выбранные версии и почему

Критерии приёмки:
- Три smoke-теста зелёные
- Package.resolved закоммичен

## [SWIFTMIG-7] Спроектировать доменную модель
> estimate: 2д
> labels: core, arch

Определить core-сущности, через которые общаются все слои. Самая важная задача эпика. Все типы - value types, Sendable, без логики I/O.

Что сделать:
- AnalyzedProject (путь, тип сборки SPM/xcodeproj, модули)
- Module (имя, файлы, зависимости по именам)
- SourceFile (путь, принадлежность модулю)
- Diagnostic (код, сообщение, severity, SourceLocation)
- MigrationIssue (диагностика + категория + маршрут: deterministic(FixRuleID) / llmAssisted / manual)
- FixProposal (issue, diff/патч, статус: proposed/applied/rejected/failed)
- Enum IssueCategory: 4 категории из scope диплома + unknown
- Doc-комментарии на все публичные типы (пойдут в текст диплома)

Критерии приёмки:
- Модель компилируется в Swift 6 language mode без warnings
- Юнит-тест на Codable round-trip

## [SWIFTMIG-8] Определить публичные протоколы слоёв (порты архитектуры)
> estimate: 1д
> labels: core, arch

Зафиксировать контракты между подсистемами (ports & adapters): core объявляет протоколы, реализации приходят позже. Позволяет разрабатывать этапы 1-3 независимо и мокать в тестах.

Что сделать:
- ProjectLoader: путь -> AnalyzedProject
- DiagnosticsProvider: проект/модуль -> [Diagnostic]
- IssueClassifier: [Diagnostic] + AST-контекст -> [MigrationIssue]
- FixRule: применимость + трансформация для детерминированных фиксов
- LLMProvider: запрос с контекстом -> структурированный ответ
- Мок для каждого протокола в тестовом таргете
- Диаграмма зависимостей протоколов в docs/architecture.md

Критерии приёмки:
- Все протоколы имеют моки
- Ни один протокол не ссылается на конкретные реализации
- Диаграмма закоммичена

## [SWIFTMIG-9] Скелет MigratorCLI
> estimate: 0.5д
> labels: cli

CLI - рабочая лошадка диплома: через него пойдут batch-прогоны корпуса проектов на этапе эксперимента. Каркас команд на ArgumentParser с заглушками.

Что сделать:
- Команды: analyze <path> (печатает "found N modules" на мок-данных), report <path> --format json|md (заглушка)
- Вывод в stdout + машиночитаемый JSON по флагу --json
- Коды возврата: 0 успех, 1 ошибка анализа, 2 невалидный вход

Критерии приёмки:
- swift run migrator analyze ./Fixtures/DemoProject отрабатывает на фикстуре
- --help описывает все команды

## [SWIFTMIG-10] Скелет MigratorUI
> estimate: 2д
> labels: ui

Минимальное SwiftUI-приложение: выбор проекта, передача в core, отображение результата-заглушки. Цель - проверить связку UI и core и зафиксировать паттерн (MV с @Observable или MVVM, решение в DECISIONS.md).

Что сделать:
- Окно с кнопкой "Открыть проект" -> NSOpenPanel (выбор папки)
- AppState / root-модель с DI core-сервисов через протоколы (в превью и тестах - моки)
- Экран-заглушка "Проект загружен: N модулей" на мок-данных
- Sandbox: выключить или настроить security-scoped bookmarks для доступа к выбранной папке - исследовать и записать решение

Критерии приёмки:
- Приложение открывает папку и показывает данные от мок-loader'а
- Превью работают без реального core

## [SWIFTMIG-11] CI: сборка и тесты
> estimate: 1д
> labels: infra

GitHub Actions на macOS-раннере: сборка пакета, тесты, сборка app-таргета. Зелёный CI с первого дня + скриншот в главу "организация разработки".

Что сделать:
- Workflow на push/PR: swift build, swift test, xcodebuild -scheme MigratorUI build
- Кэш .build и SPM-зависимостей (swift-syntax без кэша собирается 15+ минут)
- Бейдж статуса в README

Критерии приёмки:
- CI зелёный на main
- Повторный прогон с кэшем заметно быстрее первого
